Sujet : Les Avancées dans les Modèles de Langage Naturel

Les avancées dans les modèles de langage naturel font référence aux progrès réalisés dans le domaine de l'intelligence artificielle, en particulier dans le développement de modèles de traitement du langage naturel (NLP). Les modèles de langage sont comme des robots qui apprennent en regardant beaucoup de textes.

Ces avancées incluent l'utilisation de modèles neuronaux tels que les réseaux de neurones profonds, en particulier les transformers (une sorte de superordinateur qui aide à comprendre et à créer du langage écrit de manière vraiment intelligente, en regardant tout le texte à la fois au lieu de le lire mot par mot.), qui ont montré une capacité exceptionnelle à comprendre et à générer du langage humain.

Ces intelligences artificielles utilisent le machine learning pour résoudre des problèmes sans nécessiter de coder toutes les règles manuellement.

Elles apprennent à partir de données labellisées, comme des avis positifs ou négatifs sur des films, pour déterminer automatiquement les règles permettant de résoudre le problème. En gros, on leur montre le problème et elles trouvent elles-mêmes comment le résoudre en se basant sur les données fournies.

Ces étiquettes sont utilisées pour entraîner des modèles d'apprentissage automatique à reconnaître et à prédire des informations similaires sur de nouvelles données non labellisées.

Points positifs :

Les modèles de langage peuvent automatiquement apprendre à partir de données et résoudre des problèmes complexes.

Le deep learning, en particulier avec les réseaux récurrents, améliore la performance sur le texte et les images.

Points négatifs :

Les modèles de langage peuvent présenter des lacunes telles que des erreurs factuelles, des hallucinations et des problèmes d'alignement.

Les données utilisées peuvent devenir obsolètes, surtout avec des modèles volumineux, ce qui rend la mise à jour difficile.

Pas encore très bon en maths et en arithmétique

Info :

Les hallucinations dans le contexte des modèles de langage se produisent lorsque le modèle génère du texte qui semble être correct mais qui est en réalité faux ou incorrect.

Les problèmes d'alignement se réfèrent à la désynchronisation entre les différentes parties du texte généré par le modèle. Par exemple, le modèle peut prédire correctement les premiers mots d'une phrase mais perdre le fil par la suite, conduisant à une réponse qui semble incohérente ou déconnectée. Cela peut rendre la sortie du modèle moins précise ou difficile à interpréter.

Avancées futures :

Connexion à des outils externes via des frameworks comme ReAct :

Les modèles de langage peuvent être connectés à des outils externes tels que des bases de données de connaissances (comme Wikipedia) ou d'autres APIs.

Ces outils externes peuvent enrichir les connaissances du modèle en lui fournissant des informations supplémentaires pour améliorer ses réponses.

Par exemple, si le modèle ne connaît pas une réponse à une question donnée, il peut faire une requête à Wikipedia pour obtenir la réponse et la fournir à l'utilisateur.

Si on a besoin de faire un calcule, ou de l'arithmetique, on peux lui demander d'utiliser une calculatrice.

Approches telles que Toolformer pour un ajustement fin dans l'utilisation des APIs :

Toolformer propose une méthode d'ajustement fin pour guider le modèle dans l'utilisation des APIs externes.

Plutôt que de laisser le modèle générer lui-même les appels d'API, l'utilisateur peut intervenir pour superviser et ajuster ces appels.

Cela permet de limiter les erreurs en contrôlant plus précisément les sources d'information utilisées par le modèle.

Par exemple, l'utilisateur peut intervenir pour choisir quelle API utiliser pour une tâche spécifique ou comment intégrer les résultats de l'API dans la réponse générée par le modèle.
